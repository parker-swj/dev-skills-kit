# /learn — 提取会话经验到知识库

在解决了非平凡问题后运行 `/learn`，将本次会话的经验沉淀为可复用知识。

## 触发时机

- 解决了一个棘手的 bug 或技术难题
- 发现了项目特定的规律或约定
- 找到了某个库/API 的使用技巧或坑
- 完成了一个有参考价值的实现模式

## 执行步骤

1. **回顾会话**：扫描当前会话，识别有价值的模式

2. **判断可提取性**：以下内容值得保存

   | 类型 | 描述 |
   |------|------|
   | 错误解决模式 | 具体错误的根因与修复方式 |
   | 调试技巧 | 非显而易见的调试步骤或工具组合 |
   | 绕过方案 | 库/框架的 quirks 和 workaround |
   | 项目约定 | 发现的代码库约定或架构决策 |

3. **跳过不值得保存的内容**
   - 简单 typo 修复
   - 一次性问题（特定 API 宕机等）
   - 显而易见的基础知识

4. **调用保存工具**（支持双写与本地降级）：
   - **检查环境**：判断是否支持 MCP `add_knowledge` 工具，以及用户/部署配置是否要求本地保存（如指定了 `Knowledge_Mode: local`）。
   - **全局保存（MCP）**：若 MCP 工具可用，通过工具写入中心化知识库：
     - `category`：`experience`（首选）或 `solution`
     - `title`：简洁描述性标题（中文）
     - `tags`：相关技术标签
     - `content`：包含「问题」「解决方案」「适用场景」三段
   - **本地保存（Fallback / 双写）**：
     - 若 MCP 不可用或明确要求本地优先，将知识格式化为 Markdown 文件（含 YAML frontmatter），保存到本项目的 `.agent/knowledge/<kebab-case-title>.md` 中。

5. **向用户确认**：保存后告知用户保存了什么内容，以及**保存的位置**（中心化知识库 ID，或本地文件路径）。

## 知识条目格式模板

```markdown
## 问题

[具体描述遇到的问题，包含足够上下文]

## 解决方案

[具体的解决步骤或代码]

## 适用场景

[何时会遇到这个问题，有哪些前提条件]

## 注意事项（可选）

[坑点、副作用、相关限制]
```

## 与 /save-to-kb 的区别

| | /learn | /save-to-kb |
|--|--------|------------|
| 触发方式 | 主动调用，总结会话经验 | 保存任意知识片段 |
| 内容来源 | 从会话中自动提取 | 用户明确指定内容 |
| 典型用途 | 会话结束前沉淀经验 | 随时保存某个发现 |
